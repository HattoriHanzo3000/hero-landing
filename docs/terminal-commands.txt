Hero Landing — useful terminal commands
=========================================

Run these from the project folder (hero-landing). In Cursor: Terminal → New Terminal.
If you are elsewhere, go to the project first:

  cd /Users/Ildar/hero-landing


--- First time (or after pulling new dependencies) ---

  npm install

  Downloads packages listed in package.json into node_modules.
  Do this once per machine, or when package.json / package-lock.json changed.


--- Day to day: run the site locally ---

  npm run dev

  Starts the Next.js development server. Keep this terminal open.
  Open in a browser: http://localhost:3000

  Stop the server: press Ctrl+C in that terminal.


--- Other npm scripts (this project) ---

  npm run build

  Builds an optimized production version into .next/
  Use before deploy or to check that the site compiles.


  npm run start

  Serves the production build. Run npm run build first.
  Also http://localhost:3000 by default.


  npm run lint

  Runs ESLint on the codebase to catch common code issues.

--- Git (this repo) ---

  git status

  Shows changed and untracked files.


  git diff

  Shows line-by-line changes not yet committed.


  git log --oneline -10

  Last 10 commits (short view).


  git pull

  Fetch and merge latest from the remote branch.


  git add path/to/file

  Stage a file for commit (replace with real path).


  git commit -m "Short message describing why"

  Save staged changes as a commit (only when you intend to commit).


--- Handy general terminal commands ---

  pwd

  Print current directory (where the terminal is “standing”).


  ls

  List files and folders in the current directory.


  ls -la

  List including hidden files and details.


  cd folder-name

  Enter a subfolder. Example: cd docs


  cd ..

  Go up one folder.


  clear

  Clear the terminal screen (history is still there).


  node -v
  npm -v

  Check Node.js and npm versions (need Node installed for this project).


--- If port 3000 is already in use (macOS) ---

  lsof -i :3000

  Shows which process is using port 3000.


  kill -9 <PID>

  Replace <PID> with the number from lsof. Stops that process.
  Then run npm run dev again.


--- Quick reference: what npm run X means ---

  npm     = Node Package Manager (installs deps, runs scripts)
  run     = execute a named script from package.json
  dev     = script name → runs "next dev" (development server)

  See package.json → "scripts" for the full list in this project.
